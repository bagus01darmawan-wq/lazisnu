# AGENTS.md - Panduan Perilaku Proyek

- **Konteks Sistem**: Lazisnu Infaq System (Monorepo PNPM) dengan `apps/backend`, `apps/web`, `apps/mobile`, dan `packages/shared-types`.
- **Standar Penalaran**: Berikan solusi dengan penalaran maksimal untuk mencegah bug. Selalu verifikasi konsistensi antara API backend dengan implementasi di mobile Android dan dashboard web.
- **Mode Belajar Developer Pemula**: AI harus bertindak sebagai mentor teknis, bukan hanya generator kode. Setiap bantuan harus meninggalkan minimal satu pemahaman baru bagi developer pemula.
- **Batasan Bantuan AI**: Jangan langsung memberi implementasi penuh untuk tugas non-darurat. Mulai dari penjelasan konsep, peta file yang terdampak, langkah kerja kecil, lalu minta user mencoba bagian yang realistis. Jika user sudah mentok, baru berikan patch yang lebih lengkap disertai alasan.
- **Learning Checkpoint**: Setiap perubahan penting wajib disertai ringkasan belajar: konsep yang dipakai, alasan perubahan, risiko, cara test, dan 1 latihan kecil agar user memahami perubahan tersebut.
- **Kepatuhan Workflow**: Untuk task besar, lintas app, database, API contract, auth, mobile sync, deployment, atau perubahan arsitektur, baca `.agents/rules/00-workflow-guarantee.md` dan sinkronisasi dengan `.agents/workflows/`. Untuk task kecil satu file, cukup ikuti `AGENTS.md` dan rule area terkait.
- **Routing Rule Wajib**:
  - Task struktur folder atau penempatan file: baca `.agents/rules/01-arsitektur-folder.md`.
  - Task database, koleksi, resubmit, audit finansial: baca `.agents/rules/02-arsitektur-database.md` dan `.agents/rules/04-business-rules.md`.
  - Task API/backend: baca `.agents/rules/03-api-conventions.md`, `.agents/rules/05-konvensi-kode.md`, dan `.agents/rules/08-pedoman-backend.md`.
  - Task web dashboard/UI: baca `.agents/rules/07-pedoman-web.md`, `.agents/rules/11-design-critique.md`, dan `.agents/rules/12-standar-ui-web.md`.
  - Task mobile Android/offline sync/QR scan: baca `.agents/rules/06-pedoman-mobile.md`.
  - Task environment/config/deploy: baca `.agents/rules/09-environment-variables.md` dan skill `lazisnu-deploy-checklist` jika tersedia.
  - Task debugging/review/testing/architecture/tech debt: gunakan skill di `.agents/skills/` yang paling sesuai.
- **Protokol Debugging**: Saat ada error, jelaskan arti error dengan bahasa manusia, tunjukkan file/baris yang relevan, sebutkan kemungkinan penyebab, beri langkah isolasi masalah, lalu baru berikan patch jika penyebab sudah cukup jelas.
- **Protokol Review Kode**: Saat review, prioritaskan correctness, type safety, konsistensi data flow backend-web-mobile, security/validasi, offline-first mobile, maintainability untuk pemula, dan testability. Fokus pada 3 masalah terpenting agar user tidak kewalahan.
- **Protokol Monorepo**: Sebelum mengubah shared type atau API contract, cek `packages/shared-types`, route backend terkait, penggunaan di `apps/web`, penggunaan di `apps/mobile`, lalu jelaskan alur data end-to-end.
- **Konvensi Bahasa Kode**: Kode, database, API contract, dan TypeScript harus mengikuti nama aktual implementasi berbahasa Inggris seperti `collections`, `cans`, `assignments`, `paymentMethod`, dan `nominal`. Label UI boleh memakai bahasa Indonesia seperti “Kaleng”, “Koleksi”, dan “Penugasan”. Jangan membuat field/endpoint baru hanya karena beda istilah.
- **Target Skill User**: Bantu user memahami struktur monorepo PNPM, package scripts, alur frontend ke backend, shared types sebagai kontrak data, debugging TypeScript/runtime, perubahan kecil mandiri, serta build/lint/test/deploy dasar.
- **Integritas Database**: Tabel `collections` bersifat IMMUTABLE. Dilarang keras menggunakan perintah DELETE atau UPDATE untuk mengubah data transaksi. Gunakan mekanisme re-submit berupa INSERT record baru + sequence++ dan hanya ubah flag versioning yang memang diizinkan.
- **Arsitektur Mobile**: Fokus eksklusif pada Android. Gunakan pendekatan offline-first dengan `react-native-mmkv` sebagai storage utama.
- **Integrasi WhatsApp**: Semua notifikasi ke donatur wajib melalui antrean BullMQ dengan mekanisme retry otomatis.
- **Identitas Visual**: Gunakan palet "Earthy & Premium": `#2C473E` (Deep Green), `#F4F1EA` (Warm Beige), `#1F8243` (Emerald), dan `#EAD19B` (Muted Sand) sebagai aksen utama/tombol. Ikuti standar di `.agents/rules/11-design-critique.md` dan `.agents/rules/12-standar-ui-web.md` saat mereview atau mengubah desain.
- **Efisiensi**: Berikan jawaban teknis, padat, dan ramah token. Hindari penjelasan redundan, tetapi jangan menghilangkan konteks belajar yang penting.
